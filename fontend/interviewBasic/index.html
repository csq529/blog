    7.script、script async 和script defer的区别

    script HTML解析中断，脚本被提取并立即执行，执行结束后，HTML解析继续

    script async 脚本的提取、执行的过程与HTML解析过程并行，脚本执行完毕可能在HTML解析完毕之前。当脚本与页面上其他脚本独立时，可以使用async,比如用作统计分析

    script defer脚本仅提取过程与HTML解析过程并行，脚本的执行将在HTML解析完毕后进行。如果有多个含有defer的脚本，脚本的执行顺序将按照document中出z的位置，从上到下顺序执行。

    3.css定位 1.静态定位：通过设置position属性为static，元素会按照正常的文档流进行布局，不会脱离文档流 2.相对定位：通过设置position属性为relative，元素会按照正常的文档流进行布局，但会脱离文档流，可以通过top、right、bottom、left属性来设置元素的位置，相对于元素在文档流的原始位置进行定位 3.绝对定位：通过设置position属性为absolute，元素会脱离文档流，并固定在指定位置，相对于其最近已定位（非static定位）的祖先元素进行定位，如果没有这样的祖先元素，则相对于文档的html元素进行定位 4.固定定位：通过设置position属性为fixed，元素会脱离文档流，并固定在指定位置，相对于浏览器视口进行定位 5.粘性定位：通过设置position属性为sticky，可以认为是相对定位和固定定位的结合体，元素在页面滚动到某个阈值之前是按照正常文档流进行布局，当到达这个阈值时，它的表现就像固定定位一样

    4.雪碧图 雪碧图是一种将多个图片合并成一张图片的方式，从而减少http请求，提高页面加载速度，通过css的background-image属性来设置图片，通过background-position属性来设置图片的位置

    行内元素垂直居中：line-height: 元素高度; 块级元素垂直居中：子绝父相，设置子元素top: 50%,元素高度不确定使用transform: translateY(-50%),否则可以使用 margin: margin-top: -元素高度的一半 flexbox：display: flex; align-items: center;

    水平垂直居中：宽高确定： 1、子绝父相, left: 0 right: 0; top: 0 bottom: 0; margin: auto; 2、子绝父相, left: 50%; top: 50%; margin-left: -元素宽度的一半; margin-top: -元素高度的一半;

    水平垂直居中 宽高不确定： 1、子绝父相, left: 50%; top: 50%; transform: translate(-50%, -50%); 2、flexbox：display: flex; justify-content: center; align-items: center;

    6.BFC 是块级格式化上下文，它是一个独立渲染的区域，只有Black-level Box参与，它规定了内部的Black-level Box如何布局，并且与外部毫不相干 BFC布局规则： 1、内部的box会在垂直方向，一个接一个的放置 2、Box的垂直方向的距离由margin决定，属于同一个BFC的两个相邻的box会发生重叠 3、每个元素的margin box的左边，与包含块border box的左边相接触（对于从左往右的格式化，否则相反），即使浮动元素也是如此。 4、BFC的区域不会与float box重叠 5、BFC是页面上一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之如此。 6、计算BFC的高度时，浮动元素也参与计算 作用： 1、自适应两栏布局 2、清除内部浮动 3、放置垂直的margin重叠

    2. this this是执行上下文的一个属性，它指向最后一个调用这个方法的对象，在实际开发中，this的指向可以通过四种调用模式来判断 1、函数调用模式：当一个函数不是一个对象的属性时，直接作为函数来调用时，this指向全局对象 2、方法调用模式：如果一个函数作为一个对象的属性调用时，this指向这个对象 3、构造器模式调用：如果一个函数用new调用时，函数执行前会创建一个对象，this指向这个新创建的对象 4、apply、call、bind调用模式：这三个方法都可以显示的指定调用函数的this指向。 其中apply的方法接受两个参数：一个是this绑定的对象，一个是参数数组. call方法接受的参数，第一个是this绑定的对象，后面的其余参数是传入函数执行的参数。也就是说，在使用call方法时，传递给函数的参数是逐个列出来的。bind方法通过传入一个对象，返回一个this绑定了传入对象的新函数。这个函数的this指向除了使用new时，会发生变化，其他情况下都不会改变。 优先级：构造器 > apply、call、bind > 方法 > 函数

    3. AMD和CommonJS的区别 都是实现模块体系的方式，CommonJs是同步的，而AMD是异步的，CommonJs的设计是为服务器端开发考虑的，而AMD支持异步加载模块，更适合浏览器 AMD的语法非常冗长，CommonJs更接近其他语言import声明语句的语法习惯，更接近Node编写模块的风格

    4. ES6和CommonJS的区别 CommonJS是对模块的浅拷⻉，ES6 Module是对模块的引⽤，即ES6 Module只存只读，不能改变其值，也就是指针指向不能变，类似const； import的接⼝是read-only（只读状态），不能修改其变量值。 即不能修改其变量的指针指向，但可以改变变量内部指针指向，可以对commonJS对重新赋值（改变指针指向），但是对ES6 Module赋值会编译报错。

    5. var、let、const 1、块级作用域：由{}包括，let和const具有块级作用域，var不存在，块级作用域解决了ES5中的两个问题：内层变量可能覆盖外层变量，用来继续循环变量泄漏为全局变量 2、变量提升：var存在变量提升，let、const不存在，即只能在变量声明之后使用，否则会报错 3、给全局添加属性：浏览器的全局对象是window,Node的全局对象是global,var声明的变量为全局变量，并且将该变量添加为全局对象的属性，但是let和const不会 4、重复声明：var可以,let和const不能 5、暂时性死区：在使用let、const声明变量之前，该变量都是不可用的，在语法上，成为暂时性死区，var不存在 6、初始值设置：在变量声明时，var和let可以不设置初始值，但const必须设置初始值 7、指针指向：let和const都是ES6新增的，let可以更改指针指向（可以重新赋值），但const声明的变量是不允许改变指针的指向 区别 var let const 是否有块级作用域 × ✔️ ✔️ 是否存在变量提升 ✔️ × × 是否添加全局属性 ✔️ × × 能否重复声明变量 ✔️ × × 是否存在暂时性死区 × ✔️ ✔️ 是否必须设置初始值 × × ✔️ 能否改变指针指向 ✔️ ✔️ ×

    8. for in 和 for of的区别

    for of 遍历获取的是对象的键值，for in获取的是对象的键名 for in 会遍历对象的整个原型链，性能非常差，不推荐使用，而for of只遍历当前对象不会遍历原型链 对于数组的遍历，for in 会返回数组中所有可枚举的属性（包括原型链上可枚举的属性）for of 只返回数组的下标对应的属性值 for in 主要是为遍历对象而生，不适用遍历数组，for of循环用来遍历数组、类数组对象、字符串、Set、Map以及Generator对象

    2. 原型和原型链 JavaScript的原型是实现对象继承的核心机制，原型链是通过proto连接的属性查找链条，所有的对象最终继承自Object.prototype，终点是null 原型和__proto__的区别： 1、prototype（显示原型）：只有函数对象才拥有该属性，指向一个原型对象： 构造函数通过prototype属性定义实例共享的属性和方法 原型对象包含constructor属性，指回构造函数 2、proto(隐示原型)：每个对象（除了null外）都拥有改属性，指向改对象的原型对象 实例对象的__proto__指向构造函数的prototype ES6标准推荐使用Object.getPrototypeOf()替代__proto__

    原型链的工作原理： 1、属性查找机制：当访问一个对象属性时，JavaScript引擎按照以下顺序查找 先在对象自身查找属性 若找不到，沿__proto__向上查找原型对象 继续查找原型的原型，直到object.prototype 若仍未找到，返回undefined 2、原型链结构： 实例对象 -> Person.prototype -> Object.prototype -> null 所有内置对象（Array、String、Function等）的原型链最终指向Object.prototype Function.prototype.proto === Object.prototype

    2. 异步编程 JavaScript中的异步机制可以分为以下几种：

      • 回调函数 的方式，使用回调函数的方式有一个缺点是，多个回调函数嵌套的时候会造成回调函数地狱，上下两层的回调函数间的代码耦合度太高，不利于代码的可维护。
      • Promise 的方式，使用 Promise 的方式可以将嵌套的回调函数作为链式调用。但是使用这种方法，有时会造成多个 then 的链式调用，可能会造成代码的语义不够明确。
      • generator 的方式，它可以在函数的执行过程中，将函数的执行权转移出去，在函数外部还可以将执行权转移回来。当遇到异步函数执行的时候，将函数执行权转移出去，当异步函数执行完毕时再将执行权给转移回来。因此在 generator 内部对于异步操作的方式，可以以同步的顺序来书写。使用这种方式需要考虑的问题是何时将函数的控制权转移回来，因此需要有一个自动执行 generator 的机制，比如说 co 模块等方式来实现 generator 的自动执行。 -async 函数 的方式，async 函数是 generator 和 promise 实现的一个自动执行的语法糖，它内部自带执行器，当函数内部执行到一个 await 语句的时候，如果语句返回一个 promise 对象，那么函数将会等待 promise 对象的状态变为 resolve 后再继续向下执行。因此可以将异步逻辑，转化为同步的顺序来书写，并且这个函数可以自动执行。

    3. 事件循环 JS是单线程，防止代码阻塞，把任务分为同步和异步 同步代码给JS引擎执行，异步代码交给宿主环境（浏览器、Node） 同步代码放入执行栈，异步代码等待时机成熟送入任务队列排队 执行栈执行完毕，回去任务队列查看是否有异步代码，有就送到执行栈执行，反复循环查看执行这个过程就是事件循环（EventLoop）

    宏任务和微任务 宏任务：宿主环境发起（浏览器、Node） 常见：script 事件、网络请求、定时器、动画、IO 微任务：JS引擎发起的任务 Promise本身是同步，then/catch的回调函数是异步的 process.nextTick(Node)，Promise.then() Promise.catch() async/await 执行过程： 同步任务 微任务的异步代码 宏任务的异步代码
